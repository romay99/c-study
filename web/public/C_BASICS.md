# C 기초 용어 – Java만 해본 사람을 위한 설명

C를 처음 보면 `#include`, `#define`, `#ifndef` 같은 게 낯설 수 있습니다.  
Java에 대비해서 **무슨 뜻인지**, **왜 쓰는지**만 간단히 정리했습니다.

---

## 1. `#include` – 다른 파일 가져오기

```c
#include <stdio.h>
#include "basics.h"
```

- **의미:** "이 파일의 내용을 여기에 그대로 붙여 넣어라."
- **Java와 비교:**  
  - Java의 `import java.util.List;` 처럼 **다른 코드를 쓸 수 있게 불러오는 것**이라고 보면 됩니다.  
  - 다만 Java는 "이름만 알려주는" 느낌이고, C는 **파일 전체 내용을 복붙**한다고 생각하면 됩니다.
- **`<stdio.h>` vs `"basics.h"`**
  - `<stdio.h>`: **표준 라이브러리** (컴파일러가 정해진 경로에서 찾음).  
    → Java의 `java.lang`, `java.util` 같은 "기본 제공 라이브러리".
  - `"basics.h"`: **우리 프로젝트 안의 파일** (현재 폴더나 지정한 include 경로에서 찾음).  
    → Java의 우리가 만든 패키지/클래스 파일.

---

## 2. `#define` – 이름을 정해 두기 (매크로)

```c
#define MAX_SIZE 100
#define BASICS_H
```

- **의미:** "앞으로 이 이름을 만나면, 그 자리를 저 값/빈칸으로 바꿔라."
- **Java와 비교:**
  - `#define MAX_SIZE 100`  
    → Java의 `public static final int MAX_SIZE = 100;` 처럼 **상수**로 쓰는 경우가 많습니다.  
    (C에는 `final`이 없어서, 상수처럼 쓰려면 보통 `#define`이나 `const`를 씀.)
  - `#define BASICS_H`  
    → 값은 안 정하고 그냥 "이 이름이 정의되었다"만 표시할 때 씁니다.  
    → 아래의 `#ifndef`와 같이 써서 "이미 한 번 읽은 헤더인가?" 체크할 때 사용.

---

## 3. `#ifndef` / `#endif` – "아직 정의 안 됐을 때만 실행"

```c
#ifndef BASICS_H
#define BASICS_H
/* ... 헤더 내용 ... */
#endif
```

- **의미:**
  - `#ifndef BASICS_H`  
    → "BASICS_H가 **아직 정의되지 않았으면** (#if n def = if not defined)"  
    → 조건이 참이면 아래부터 `#endif`까지를 컴파일에 포함.
  - `#define BASICS_H`  
    → "이제 BASICS_H를 정의했다"고 표시.  
    → 다음에 같은 헤더가 다시 `#include`되면, `#ifndef BASICS_H`가 거짓이 되어서 안쪽을 **다시 읽지 않음**.
  - `#endif`  
    → `#ifndef`로 시작한 블록의 끝.

- **왜 쓰나?**  
  C에서는 `#include`가 **파일 내용을 그대로 붙여 넣기**라서,  
  같은 헤더를 여러 번 include하면 **같은 선언이 두 번 나와서 컴파일 에러**가 납니다.  
  그래서 "이 헤더는 한 번만 실제로 쓰이게 하자"라고 **문을 하나 두는 것**이 `#ifndef` ~ `#define` ~ `#endif`입니다.  
  이걸 **include guard(헤더 가드)** 라고 부릅니다.

- **Java와 비교:**  
  Java에는 이런 게 없습니다.  
  Java는 `import`로 클래스 이름만 알려주고, 같은 클래스를 두 번 써도 한 번만 로드되기 때문입니다.

---

## 4. 주석 – `/* */` 와 `//`

```c
/* 이렇게 하면 여러 줄 주석 */
// 이렇게 하면 그 줄 끝까지 주석 (Java와 동일)
```

- **Java와 비교:**
  - `/* ... */`: Java의 `/* ... */` 와 같음. 여러 줄 주석.
  - `//`: Java의 `//` 와 같음. 한 줄 주석.

---

## 5. `.h`(헤더) 와 `.c`(구현) – 파일 역할

| 확장자 | 역할 | Java로 치면 |
|--------|------|-------------|
| **.h** | 함수/구조체의 **선언**만 (이름, 인자, 반환 타입 등) | 인터페이스 메서드 목록, 또는 클래스의 public 메서드 시그니처만 있는 것 |
| **.c** | 함수의 **실제 구현** (몸통 코드) | 그 인터페이스/클래스의 실제 구현부 |

- **왜 나누나?**  
  - 여러 `.c` 파일에서 같은 함수를 쓰려면, "이런 함수가 있다"는 **선언**이 필요합니다.  
  - 그 선언을 한곳에 모아 둔 게 `.h`이고,  
  - "그 함수가 정말로 뭘 하는지"는 `.c`에만 씁니다.  
  → Java의 "인터페이스 vs 구현 클래스" 분리와 비슷한 느낌입니다.

---

## 6. `void` – "없음"

```c
void learn_basics(void);
```

- **의미:**
  - 반환 타입 자리의 `void`: "이 함수는 **아무것도 반환하지 않는다**."  
    → Java의 `void` 메서드와 같음.
  - 괄호 안의 `void`: "이 함수는 **인자(파라미터)가 없다**."  
    → Java에서는 그냥 `()` 로 두면 되는데, C에서는 "인자 없음"을 분명히 하고 싶을 때 `(void)` 라고 쓰는 경우가 있습니다.  
    (그냥 `()`만 써도 "인자 없음"으로 쓰는 경우가 많음.)

---

## 7. `;` (세미콜론)

- C에서는 **문장 끝에 반드시 `;`** 를 붙입니다.  
- Java와 같다고 보면 됩니다.

---

## 8. 정리 표

| C에서 보는 것 | 의미 (한 줄) | Java로 치면 |
|---------------|--------------|-------------|
| `#include <파일>` | 표준 라이브러리 파일 가져오기 | `import java.xxx.*` (기본 라이브러리) |
| `#include "파일"` | 우리가 만든 파일 가져오기 | 우리 패키지/클래스 import |
| `#define 이름 값` | 이름을 값으로 치환 (상수·매크로) | `static final` 상수 등 |
| `#define 이름` | "이 이름 정의됨" 표시 (가드용) | (없음) |
| `#ifndef 이름` | "이 이름이 아직 없으면" | (없음) |
| `#endif` | 조건부 블록 끝 | (없음) |
| `/* */` | 여러 줄 주석 | `/* */` |
| `//` | 한 줄 주석 | `//` |
| `.h` | 선언만 모아 둔 파일 | 인터페이스/시그니처만 있는 것 |
| `.c` | 실제 구현이 있는 파일 | 구현 클래스 |
| `void` (반환) | 반환값 없음 | `void` |
| `void` (인자) | 인자 없음 | `()` |

---

이제 `include/basics.h` 같은 파일을 열면, 맨 위에 있는 `#ifndef BASICS_H` 같은 게  
"이 헤더가 두 번 포함되지 않도록 하는 문"이라고 이해하시면 됩니다.  
더 자세한 건 각 `.h` 파일 맨 위에 적어 둔 주석을 참고하면 됩니다.
